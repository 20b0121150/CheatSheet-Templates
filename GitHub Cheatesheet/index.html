This GitHub Cheatsheet is for you.!
  • Configuring user information used across all local repositories: set a name that is identifiable for credit when review version history 
    git config --global user.name “[firstname lastname]”
  • Configuring user information used across all local repositories: set an email address that will be associated with each history marker 
    git config --global user.email “[valid-email]”
  • Configuring user information used across all local repositories: set automatic command line coloring for Git for easy reviewing 
    git config --global color.ui auto
  • Configuring user information, initializing and cloning repositories: initialize an existing directory as a Git repository 
    git init
  • Configuring user information, initializing and cloning repositories: retrieve an entire repository from a hosted location via URL 
    git clone [url]
  • Working with snapshots and the Git staging area: show modified files in working directory, staged for your next commit 
    git status
  • Working with snapshots and the Git staging area: add a file as it looks now to your next commit (stage) 
    git add [file]
  • Working with snapshots and the Git staging area: unstage a file while retaining the changes in working directory 
    git reset [file]
  • Working with snapshots and the Git staging area: diff of what is changed but not staged 
    git diff
  • Working with snapshots and the Git staging area: diff of what is staged but not yet committed 
    git diff --staged
  • Working with snapshots and the Git staging area: commit your staged content as a new commit snapshot 
    git commit -m “[descriptive message]”
  • Isolating work in branches, changing context, and integrating changes: list your branches. a * will appear next to the currently active branch 
    git branch
  • Isolating work in branches, changing context, and integrating changes: create a new branch at the current commit 
    git branch [branch-name]
  • Isolating work in branches, changing context, and integrating changes: switch to another branch and check it out into your working directory 
    git checkout
  • Isolating work in branches, changing context, and integrating changes: merge the specified branch's history into the current one 
    git merge [branch]
  • Isolating work in branches, changing context, and integrating changes: show all commits in the current branch's history 
    git log
  • Examining logs,diffs and object information: show the commit history for the currently active branch 
    git log
  • Examining logs,diffs and object information: show the commits on branchA that is not in branchB 
    git log branchB..branchA
  • Examining logs,diffs and object information: show the commits that changed file even across renames 
    git log --follow[file]
  • Examining logs,diffs and object information: show the diff of what is in branchA that is not in branchB 
    git diff branchB..branchA
  • Examining logs,diffs and object information: show any object in Git in human-readable format 
    git show [SHA]
  • Versioning file removes and path changes: delete the file from project and stage the removal for commit 
    git rm [file]
  • Versioning file removes and path changes: change an existing file path and stage the move 
    git mv [existing-path] [new-path]
  • Versioning file removes and path changes: show all commit logs with indication of any paths that moved 
    git log --stat -M
  • Preventing unintentional staging or commiting of files: Save a file with desired patterns as .gitignore with either direct string matches or wildcard globs. 
    logs/
                    *.notes
                    pattern*/
  • Preventing unintentional staging or commiting of files: system wide ignore pattern for all local repositories 
    git config --global core.excludesfile [file]
  • Retrieving updates from another repository and updating local repos: add a git URL as an alias 
    git remote add [alias] [url]
  • Retrieving updates from another repository and updating local repos: fetch down all the branches from that Git remote 
    git fetch [alias]
  • Retrieving updates from another repository and updating local repos: merge a remote branch into your current branch to bring it up to date 
    git merge [alias]/[branch]
  • Retrieving updates from another repository and updating local repos: Transmit local branch commits to the remote repository branch 
    git merge [alias] [branch]
  • Retrieving updates from another repository and updating local repos: fetch and merge any commits from the tracking remote branch 
    git pull
  • Rewriting branches, updating commits and clearing history: apply any commits of current branch ahead of specified one 
    git rebase [branch]
  • Rewriting branches, updating commits and clearing history: clear staging area, rewrite working tree from specified commit 
    git reset --hard [commit]
  • Temporarily store modified, tracked files in order to change branches: Save modified and staged changes 
    git stash
  • Temporarily store modified, tracked files in order to change branches: list stack-order of stashed file changes 
    git stash list
  • Temporarily store modified, tracked files in order to change branches: write working from top of stash stack 
    git stash pop
  • Temporarily store modified, tracked files in order to change branches: discard the changes from top of stash stack 
    git stash drop
